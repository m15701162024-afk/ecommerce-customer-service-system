## 一、系统概述

### 1.1 产品定位

全自动网店客服采购系统是一款面向电商商家的智能化客服解决方案，支持**抖音、淘宝(千牛)、小红书**三大主流电商平台，集成1688自动采购功能，实现从客户咨询到商品采购的全流程自动化。

### 1.2 核心价值

| 价值点 | 说明 | 效果 |
|--------|------|------|
| 降本增效 | AI智能客服自动回复 | 降低70%人工成本 |
| 自动化采购 | 1688货源智能匹配 | 一键完成采购 |
| 全平台覆盖 | 一套系统管理多平台 | 统一操作界面 |
| 数据驱动 | 全链路数据分析 | 辅助业务决策 |

## 三、智能客服模块

### 3.1 模块概述

智能客服模块基于AI技术，自动识别用户咨询意图，匹配知识库进行自动回复，大幅降低人工客服工作量。

**核心功能**:
- AI意图识别
- 知识库管理
- 会话管理
- 自动回复
- 转人工服务

### 3.2 会话管理

#### 3.2.1 查看会话列表

**路径**: 智能客服 → 会话管理

**列表字段说明**:
| 字段 | 说明 |
|------|------|
| 会话ID | 系统生成的唯一标识 |
| 买家信息 | 买家昵称和ID |
| 平台来源 | 抖音/淘宝/小红书 |
| 最后消息 | 最后一条消息内容 |
| 未读数 | 买家未读消息数量 |
| 状态 | 活跃/已关闭 |
| 分配客服 | 当前处理客服 |
| 创建时间 | 会话开始时间 |

#### 3.2.2 处理会话

**步骤**:
1. 在会话列表点击买家信息进入详情
2. 查看历史消息记录
3. 在输入框输入回复内容，点击发送
4. 如需转人工，点击"转人工"按钮

**快捷操作**:
- `Enter`: 发送消息
- `Ctrl+Enter`: 换行
- `@`: 快速插入快捷回复

#### 3.2.3 转人工服务

**触发条件**:
- 用户明确要求转人工
- AI无法识别意图（置信度<0.5）
- 涉及复杂售后问题

**操作步骤**:
1. 点击会话详情页"转人工"按钮
2. 选择转人工原因
3. 点击确认
4. 系统会将会话分配给在线客服

### 3.3 知识库管理

#### 3.3.1 知识库结构

```
知识库
├── 通用知识库（全店铺适用）
├── 店铺知识库（特定店铺）
└── 商品知识库（商品相关）
```

#### 3.3.2 添加知识库条目

**路径**: 智能客服 → 知识库 → 新增

**填写内容**:
| 字段 | 必填 | 说明 |
|------|------|------|
| 分类 | 是 | 问题分类，如"售前咨询" |
| 问题 | 是 | 标准问题描述 |
| 答案 | 是 | 标准答案内容 |
| 关键词 | 否 | 触发该问题的关键词，多个用逗号分隔 |
| 优先级 | 否 | 数字越大优先级越高 |

**示例**:
```
分类: 售前咨询
问题: 商品什么时候发货？
答案: 亲，一般情况下订单会在24小时内发货哦~
关键词: 发货,多久发货,什么时候发货
优先级: 10
```

#### 3.3.3 知识库优化建议

1. **定期更新**: 每周检查知识库命中率，补充高频问题
2. **关键词优化**: 收集用户实际提问方式，丰富关键词
3. **答案优化**: 根据用户反馈优化答案，提高满意度
4. **分类清晰**: 保持分类结构清晰，便于管理

### 3.4 自动回复规则

#### 3.4.1 关键词规则

**功能**: 当用户消息包含特定关键词时，自动发送预设回复

**配置步骤**:
1. 进入"自动化引擎" → "自动回复规则"
2. 点击"新增规则"
3. 选择规则类型为"关键词"
4. 填写触发关键词（支持多个）
5. 选择匹配方式（精确匹配/包含匹配）
6. 填写回复内容
7. 设置优先级和生效时间
8. 保存规则

**示例规则**:
```
规则名称: 价格咨询回复
触发关键词: 价格,多少钱,怎么卖
匹配方式: 包含匹配
回复内容: 亲，这款商品现在活动价是XXX元，非常划算哦~
优先级: 10
状态: 启用
```

#### 3.4.2 事件规则

**功能**: 当系统发生特定事件时，自动执行动作

**支持的事件类型**:
| 事件 | 触发时机 | 应用场景 |
|------|----------|----------|
| 订单创建 | 买家下单后 | 发送确认消息 |
| 订单支付 | 买家付款后 | 发送感谢消息 |
| 订单发货 | 商家发货后 | 发送物流通知 |
| 会话开始 | 新会话建立 | 发送欢迎语 |

**配置示例**:
```
规则名称: 支付成功通知
事件类型: 订单支付
动作类型: 发送消息
消息内容: 感谢您的购买，我们会尽快为您发货~
```

### 3.5 满意度评价

#### 3.5.1 评价触发

- 会话关闭后自动发送评价邀请
- 买家可在24小时内提交评价

#### 3.5.2 查看评价

**路径**: 智能客服 → 满意度评价

**评价指标**:
- 评分: 1-5星
- 标签: 回复及时、态度好、专业耐心等
- 文字评价: 买家留言

## 四、工单系统

### 4.1 模块概述

工单系统用于记录和跟踪需要处理的问题，确保每个问题都得到及时解决。

**核心概念**:
- 工单: 记录问题的单据
- SLA: 服务等级协议，规定响应和解决时效
- 工单流转: 工单从创建到关闭的处理过程

### 4.2 工单状态说明

```
待处理(PENDING) → 已分配(ASSIGNED) → 处理中(PROCESSING) → 已解决(RESOLVED) → 已关闭(CLOSED)
                      ↑                                                    ↓
                      └──────────────── 重开(REOPENED) ←────────────────────┘
```

| 状态 | 说明 | 可执行操作 |
|------|------|------------|
| 待处理 | 新创建的工单，等待分配 | 分配、编辑 |
| 已分配 | 已指定处理人，等待开始处理 | 开始处理、转派 |
| 处理中 | 正在处理中 | 解决、转派 |
| 已解决 | 问题已解决，等待关闭 | 关闭、重开 |
| 已关闭 | 工单已结束 | 重开 |
| 已重开 | 关闭后重新打开 | 分配 |

### 4.3 创建工单

#### 4.3.1 手动创建

**路径**: 工单管理 → 工单列表 → 新增

**必填字段**:
| 字段 | 说明 |
|------|------|
| 标题 | 简要描述问题 |
| 分类 | 咨询/投诉/售后/技术支持 |
| 优先级 | 紧急/高/普通/低 |
| 买家ID | 关联的买家 |

**选填字段**:
| 字段 | 说明 |
|------|------|
| 关联订单 | 相关订单号 |
| 关联会话 | 相关会话ID |
| 详细描述 | 问题详细说明 |
| 附件 | 截图等凭证 |

#### 4.3.2 从会话创建

**步骤**:
1. 在会话详情页点击"创建工单"
2. 系统自动填充买家信息和会话ID
3. 补充工单标题和描述
4. 点击保存

### 4.4 处理工单

#### 4.4.1 分配工单

**操作步骤**:
1. 在工单列表选择待处理工单
2. 点击"分配"按钮
3. 选择处理人（可搜索）
4. 选择所属团队（可选）
5. 填写备注（可选）
6. 点击确认

**自动分配规则**:
- 系统可根据分类自动分配给对应团队
- 可根据负载均衡分配给当前工作量最少的客服

#### 4.4.2 开始处理

**条件**: 工单状态为"已分配"

**操作**: 点击"开始处理"按钮，状态变为"处理中"

**注意事项**:
- 开始处理后会记录首次响应时间
- 请在SLA响应时效内开始处理

#### 4.4.3 解决工单

**条件**: 工单状态为"处理中"

**操作步骤**:
1. 点击"解决"按钮
2. 填写解决方案（必填）
3. 填写备注（可选）
4. 点击确认

**解决方案规范**:
- 清晰描述问题原因
- 说明采取的解决措施
- 如有后续跟进事项需注明

#### 4.4.4 关闭工单

**条件**: 工单状态为"已解决"

**操作步骤**:
1. 点击"关闭"按钮
2. 选择满意度评分（1-5星）
3. 填写评价反馈（可选）
4. 点击确认

#### 4.4.5 重开工单

**场景**: 已关闭的工单需要再次处理

**操作**: 点击"重开"按钮，填写重开原因

**注意**: 重开后工单状态变为"已重开"，需要重新分配

#### 4.4.6 转派工单

**场景**: 当前处理人无法处理，需要转给其他同事

**操作步骤**:
1. 点击"转派"按钮
2. 选择新的处理人
3. 选择新的团队（如需）
4. 填写转派原因
5. 点击确认

### 4.5 SLA管理

#### 4.5.1 SLA时效标准

| 优先级 | 响应时效 | 解决时效 | 说明 |
|--------|----------|----------|------|
| 紧急 | 1小时 | 4小时 | 系统故障、重大投诉 |
| 高 | 2小时 | 8小时 | 重要客户问题 |
| 普通 | 4小时 | 24小时 | 一般咨询问题 |
| 低 | 8小时 | 48小时 | 建议反馈类 |

#### 4.5.2 超时预警

**预警方式**:
- 系统消息提醒
- 邮件通知
- 工单列表红色标记

**处理要求**:
- 响应超时需立即处理并说明原因
- 解决超时需升级给主管

### 4.6 工单统计

**路径**: 工单管理 → 工单统计

**统计维度**:
| 维度 | 说明 |
|------|------|
| 状态分布 | 各状态工单数量 |
| 优先级分布 | 各优先级工单数量 |
| 分类统计 | 各分类工单数量 |
| 处理人统计 | 各客服处理工单数 |
| 时效统计 | 平均响应/解决时间 |
| 超时统计 | 超时工单数量和比例 |

## 五、售后处理

### 5.1 模块概述

售后处理模块管理买家的退款、退货、换货申请，确保售后流程规范高效。

**售后类型**:
| 类型 | 适用场景 | 流程 |
|------|----------|------|
| 仅退款 | 未发货/无需退货 | 申请→审核→退款 |
| 退货退款 | 已发货需退货 | 申请→审核→退货→退款 |
| 换货 | 商品问题换货 | 申请→审核→退货→换货发货 |

### 5.2 售后状态说明

```
仅退款流程:
待审核(PENDING) → 退款中(REFUNDING) → 已完成(COMPLETED)
                     ↓
                  已拒绝(REJECTED)

退货退款流程:
待审核(PENDING) → 已同意(APPROVED) → 退货中(RETURNING) → 已收货(RETURNED) → 退款中(REFUNDING) → 已完成(COMPLETED)
                     ↓
                  已拒绝(REJECTED)

换货流程:
待审核(PENDING) → 已同意(APPROVED) → 退货中(RETURNING) → 已收货(RETURNED) → 换货中(EXCHANGING) → 已完成(COMPLETED)
```

### 5.3 处理售后申请

#### 5.3.1 查看售后列表

**路径**: 售后处理 → 售后列表

**列表字段**:
| 字段 | 说明 |
|------|------|
| 售后单号 | 系统生成的唯一编号 |
| 原订单号 | 关联的销售订单 |
| 买家信息 | 买家昵称和ID |
| 售后类型 | 仅退款/退货退款/换货 |
| 申请金额 | 买家申请的退款金额 |
| 状态 | 当前处理状态 |
| 申请时间 | 提交申请的时间 |

#### 5.3.2 审批售后

**步骤**:
1. 在售后列表点击"处理"进入详情
2. 查看售后原因和凭证图片
3. 查看原订单信息
4. 选择审批结果：同意/拒绝
5. 如同意，确认或修改退款金额
6. 如退货退款/换货，选择退货地址
7. 填写审批备注
8. 点击确认

**审批原则**:
- 未发货订单：一般同意退款
- 质量问题：同意退货退款或换货
- 买家原因：根据店铺政策决定
- 恶意申请：拒绝并备注原因

#### 5.3.3 填写退货物流

**触发条件**: 审批同意且为退货退款/换货类型

**操作步骤**:
1. 等待买家填写退货物流
2. 在售后详情查看物流信息
3. 跟踪物流状态
4. 商品到达后确认收货

## 六、自动化引擎

**规则类型**:
- 关键词规则：消息匹配自动回复
- 事件规则：系统事件触发动作
- 定时规则：定时执行任务

### 6.2 关键词规则配置

#### 6.2.1 创建规则

**路径**: 自动化引擎 → 自动回复规则 → 新增

**配置步骤**:
1. 填写规则名称
2. 选择规则类型：关键词
3. 配置触发条件:
   - 关键词：输入触发词，多个用逗号分隔
   - 匹配方式：精确匹配/包含匹配/正则匹配
4. 配置执行动作:
   - 动作类型：回复消息/转人工/创建工单
   - 回复内容：要发送的消息
5. 设置规则属性:
   - 优先级：数字越大越优先
   - 生效时间：全天/特定时段
   - 状态：启用/禁用
6. 保存规则

#### 6.2.2 规则示例

**欢迎语规则**:
```
名称: 首次咨询欢迎语

匹配方式: 包含匹配
动作: 回复消息
内容: 亲，欢迎咨询~ 我是您的专属客服，有什么可以帮助您的吗？
优先级: 100
```

**价格咨询规则**:
```

类型: 关键词
关键词: 价格,多少钱,怎么卖
匹配方式: 包含匹配
动作: 回复消息

优先级: 90
```

### 6.4 工作流配置

| 节点 | 说明 |
|------|------|

#### 6.4.2 创建工作流

1. 填写工作流名称和描述

## 十、常见问题

### 10.1 系统登录问题

**Q: 忘记密码怎么办？**
A: 点击登录页"忘记密码"，通过手机号或邮箱重置密码。

**Q: 账号被锁定怎么办？**
A: 联系管理员解锁，或等待30分钟后自动解锁。

**Q: 登录提示"权限不足"？**
A: 联系管理员确认账号权限配置。

### 10.2 智能客服问题

**Q: AI回复不准确怎么办？**
A: 
1. 检查知识库是否有相关条目
2. 优化关键词匹配
3. 补充相似问题的不同问法

**Q: 如何查看AI回复效果？**
A: 在"智能客服" → "会话记录"中查看，可以看到AI回复和买家反馈。

**Q: 转人工后如何重新启用AI？**
A: 在会话详情点击"启用AI"按钮。

### 10.3 工单问题

**Q: 工单超时了怎么办？**
A: 
1. 立即处理该工单
2. 在备注中说明超时原因
3. 如无法解决，及时升级给主管

**Q: 误操作关闭了工单，如何重开？**
A: 在工单详情点击"重开"按钮，填写重开原因。

**Q: 如何批量分配工单？**
A: 在工单列表选择多个工单，点击"批量分配"，选择处理人。

### 10.4 售后问题

**Q: 退款金额可以修改吗？**
A: 可以，在审批时修改退款金额，但需小于等于订单金额。

**Q: 买家填错退货物流怎么办？**
A: 联系买家重新填写，或让买家在平台修改。

**Q: 退款失败怎么处理？**
A: 
1. 查看失败原因
2. 检查支付渠道状态
3. 重新发起退款或联系技术

### 10.5 自动化规则问题

**Q: 规则设置了但不生效？**
A: 
1. 检查规则状态是否为"启用"
2. 检查生效时间设置
3. 检查优先级是否被其他规则覆盖

**Q: 如何测试规则效果？**
A: 在规则列表点击"测试"，输入测试消息查看匹配结果。

### 10.6 数据问题

**Q: 数据不更新怎么办？**
A: 
1. 点击页面刷新按钮
2. 检查网络连接
3. 清除浏览器缓存

**Q: 导出数据为空？**
A: 
1. 检查查询条件是否正确
2. 确认时间范围是否有数据
3. 尝试扩大查询范围

## 十一、最佳实践

### 11.1 客服工作规范

#### 11.1.1 响应时效

| 场景 | 要求 |
|------|------|
| 首次响应 | 30秒内 |
| 常规回复 | 2分钟内 |
| 复杂问题 | 5分钟内告知处理进度 |

#### 11.1.2 沟通技巧

**DO（应该做）**:
- 使用礼貌用语："亲"、"您好"、"感谢"
- 主动确认问题："请问您是说..."
- 提供解决方案，不只说明原因
- 结束时询问是否还有其他问题

**DON'T（不应该做）**:
- 使用"不知道"、"没办法"
- 与客户争辩
- 使用过于专业的术语
- 长时间不回复

#### 11.1.3 工单处理规范

1. **及时分配**: 新工单30分钟内完成分配
2. **规范填写**: 解决方案要清晰完整
3. **及时关闭**: 问题解决后及时关闭工单
4. **超时升级**: 即将超时时及时升级

### 11.2 售后处理原则

#### 11.2.1 快速响应

- 售后申请2小时内完成首次审核
- 紧急情况（如投诉）30分钟内响应

#### 11.2.2 合理判断

- 质量问题：优先保障买家权益
- 买家原因：按店铺政策处理
- 恶意申请：保留证据，合理拒绝

#### 11.2.3 降低损失

- 尽量协商部分退款替代全额退款
- 引导换货替代退货
- 提供优惠券替代现金补偿

### 11.3 自动化规则优化

#### 11.3.1 规则设计原则

1. **高频优先**: 优先覆盖高频问题
2. **精准匹配**: 关键词要精准，避免误触发
3. **人性化**: 回复内容要自然友好
4. **定期优化**: 根据效果数据持续优化

#### 11.3.2 规则维护周期

| 周期 | 工作内容 |
|------|----------|
| 每日 | 查看规则执行日志，处理异常 |
| 每周 | 分析规则命中率，优化低效规则 |
| 每月 | 全面review规则，补充新问题 |

### 11.4 客户运营建议

#### 11.4.1 分层运营

| 客户层级 | 运营策略 |
|----------|----------|
| 钻石会员 | 专属客服、优先发货、专属优惠 |
| 金卡会员 | 专属客服、生日礼品 |
| 银卡会员 | 优先响应、积分加倍 |
| 普通会员 | 标准服务、引导升级 |

#### 11.4.2 沉默客户唤醒

**识别标准**: 30天未购买

**唤醒策略**:
1. 发送专属优惠券
2. 推送新品信息
3. 限时折扣活动

### 11.5 数据驱动决策

#### 11.5.1 每日必看指标

- 待审核售后数量
