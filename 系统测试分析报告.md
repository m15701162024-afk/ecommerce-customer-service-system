## 二、核心业务概念定义

### 2.1 领域模型

#### 2.1.1 订单领域 (Order Domain)

```
实体: Order (订单)
├─ 标识: orderNo (订单编号)
├─ 状态: status (PENDING → PAID → PREPARING → SHIPPED → DELIVERED → COMPLETED)
├─ 金额: totalAmount, payAmount, freightAmount, discountAmount
├─ 关联: shopId, buyerId, platform, platformOrderId
├─ 时间: createdAt, paidAt, shippedAt, completedAt
└─ 业务规则:
   - 状态流转不可逆（除退款流程）
   - 支付金额 = 订单金额 + 运费 - 优惠金额
   - 取消后不能恢复
```

#### 2.1.2 工单领域 (Work Order Domain)

```
实体: WorkOrder (工单)
├─ 标识: orderNo (工单编号，格式: WO + 日期 + 随机码)
├─ 状态: status (PENDING → ASSIGNED → PROCESSING → RESOLVED → CLOSED)
├─ 优先级: priority (URGENT/HIGH/NORMAL/LOW)
├─ SLA: slaResponseTime (响应截止时间), slaResolveTime (解决截止时间)
├─ 关联: sessionId, orderId, buyerId, assigneeId
└─ 业务规则:
   - 只有PENDING/REOPENED状态可分配
   - 只有ASSIGNED状态可开始处理
   - 只有PROCESSING状态可解决
   - 只有RESOLVED状态可关闭
   - 只有CLOSED状态可重开
   - 重开后重新计算SLA时效
```

#### 2.1.3 售后领域 (After-Sale Domain)

```
实体: AfterSaleOrder (售后单)
├─ 标识: afterSaleNo (售后单号，格式: AS + 时间戳 + 随机码)
├─ 类型: type (REFUND_ONLY / RETURN_REFUND / EXCHANGE)
├─ 状态: status (PENDING → APPROVED → RETURNING → RETURNED → REFUNDING/COMPLETED)
├─ 金额: refundAmount (退款金额)
├─ 关联: orderId, buyerId, shopId
└─ 业务规则:
   - 一个订单只能有一个进行中的售后
   - 仅退款: 审批通过直接退款
   - 退货退款: 需填写物流 → 确认收货 → 退款
   - 换货: 需填写物流 → 确认收货 → 换货发货
   - 只有PENDING状态可取消
```

#### 2.1.4 采购领域 (Purchase Domain)

```
实体: PurchaseOrder (采购订单)
├─ 标识: orderNo (采购单号，格式: PO + 时间戳 + 随机码)
├─ 状态: status (AUTO_PAID / CONFIRMING / WARNING / COMPLETED / CANCELLED / REJECTED)
├─ 金额: totalAmount = unitPrice × quantity
├─ 阈值规则:
   - < 1000元: 自动支付 (AUTO_PAID)
   - 1000-10000元: 需人工确认 (CONFIRMING)
   - > 10000元: 预警状态 (WARNING)
└─ 业务规则:
   - 自动支付订单立即完成
   - 确认/预警状态需人工审批
   - 已完成订单不能取消
```
