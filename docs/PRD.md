# 电商客服系统 v2.0 产品需求文档 (PRD)

> 版本: v2.0.0 | 更新日期: 2026-03-24

---

## 一、产品概述

### 1.1 产品定位

全自动网店客服采购系统是一款面向电商商家的智能化客服解决方案，支持抖音、淘宝(千牛)、小红书三大主流电商平台，集成1688自动采购功能，实现从客户咨询到商品采购的全流程自动化。

### 1.2 目标用户

| 用户群体 | 用户特征 | 核心需求 |
|----------|----------|----------|
| 小型电商商家 | 1-5人团队，日订单<100 | 降低人力成本，自动化客服回复 |
| 中型电商商家 | 5-20人团队，日订单100-1000 | 提升客服效率，智能采购 |
| 大型电商企业 | 20+人团队，日订单>1000 | 全流程自动化，数据分析决策 |

### 1.3 核心价值

1. **降本增效** - AI智能客服降低70%人工客服成本
2. **自动化采购** - 1688货源匹配，一键采购
3. **全平台覆盖** - 一套系统管理多平台店铺
4. **数据驱动** - 全链路数据分析，辅助决策

## 二、功能模块

### 2.1 智能客服模块

#### 2.1.1 AI意图识别

**功能描述**：自动识别用户咨询意图，匹配最佳回复策略

**支持的意图类型**：
| 意图 | 说明 | 示例 |
|------|------|------|
| 商品咨询 | 商品信息、库存、规格 | "这个有货吗？" |
| 订单查询 | 订单状态、物流 | "我的订单到哪了？" |
| 价格咨询 | 价格、优惠、活动 | "能便宜点吗？" |
| 售后申请 | 退款、退货、换货 | "我想退货" |
| 物流查询 | 发货、快递、到货 | "什么时候发货？" |
| 投诉建议 | 服务投诉、商品投诉 | "我要投诉" |

**业务规则**：
- 置信度>0.8：自动回复
- 置信度0.5-0.8：提示确认
- 置信度<0.5：转人工客服

#### 2.1.2 知识库管理

**功能描述**：维护客服知识库，支持FAQ自动问答

**知识库类型**：
- 通用知识库：适用于所有店铺
- 店铺知识库：特定店铺的自定义问答
- 商品知识库：商品详情自动同步

**知识库字段**：
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 分类 | String | 是 | 问题分类 |
| 问题 | String | 是 | 标准问题 |
| 答案 | Text | 是 | 标准答案 |
| 关键词 | String[] | 否 | 触发关键词 |
| 优先级 | Integer | 否 | 匹配优先级 |

#### 2.1.3 会话管理

**功能描述**：管理客服会话全生命周期

**会话状态流转**：
```
新会话 → AI接待 → 处理中 → 已解决/转人工 → 已关闭
```

**SLA时效**：
- 首次响应：5分钟内
- 问题解决：24小时内
- 满意度评价：会话结束后24小时

---

### 2.2 工单系统模块

#### 2.2.1 工单创建

**功能描述**：创建工单记录需要处理的问题

**工单字段**：
| 字段 | 类型 | 说明 |
|------|------|------|
| 工单编号 | String | 自动生成，格式：WO+日期+随机码 |
| 标题 | String | 工单标题 |
| 分类 | Enum | 咨询/投诉/售后/技术支持 |
| 优先级 | Enum | 紧急/高/普通/低 |
| 状态 | Enum | 待处理/已分配/处理中/已解决/已关闭 |
| 买家ID | String | 关联的买家 |
| 会话ID | String | 关联的会话 |
| 订单ID | String | 关联的订单 |

#### 2.2.2 工单流程

**状态流转**：
```
待处理(PENDING)
    ↓ 分配
已分配(ASSIGNED)
    ↓ 开始处理
处理中(PROCESSING)
    ↓ 解决
已解决(RESOLVED)
    ↓ 关闭/重开
已关闭(CLOSED) ←→ 已重开(REOPENED)
```

#### 2.2.3 SLA时效管理

**时效配置**：
| 优先级 | 响应时效 | 解决时效 |
|--------|----------|----------|
| 紧急 | 1小时 | 4小时 |
| 高 | 2小时 | 8小时 |
| 普通 | 4小时 | 24小时 |
| 低 | 8小时 | 48小时 |

**超时预警**：
- 响应超时：红色标记 + 系统通知
- 解决超时：升级通知 + 自动转派

### 2.3 售后处理模块

#### 2.3.1 售后类型

| 类型 | 说明 | 流程 |
|------|------|------|
| 仅退款 | 未发货/无需退货 | 申请→审核→退款 |
| 退货退款 | 已发货需退货 | 申请→审核→退货→退款 |
| 换货 | 商品问题换货 | 申请→审核→退货→换货发货 |

#### 2.3.2 售后原因配置

**退款原因**：
- 不想要了/拍错了
- 其他平台更便宜
- 商品质量问题
- 发错货/漏发
- 商品与描述不符
- 商品破损

**换货原因**：
- 尺码不合适
- 颜色/款式不喜欢
- 商品有瑕疵

#### 2.3.3 退款流程

```
买家申请售后
    ↓
商家审核 (可调整退款金额)
    ↓ 同意
买家填写退货物流 (退货退款)
    ↓
商家确认收货
    ↓
系统执行退款
    ↓
通知买家退款成功
```

### 2.4 自动化引擎模块

#### 2.4.1 规则类型

| 类型 | 触发方式 | 应用场景 |
|------|----------|----------|
| 关键词规则 | 消息匹配 | 自动回复、FAQ响应 |
| 事件规则 | 系统事件 | 订单通知、欢迎语 |
| 定时规则 | Cron表达式 | 促销提醒、催付提醒 |

#### 2.4.2 规则配置

**关键词规则**：
```json
{
  "ruleType": "KEYWORD",
  "triggerConfig": {
    "keywords": ["价格", "多少钱", "优惠"],
    "matchType": "FUZZY"  // EXACT/FUZZY
  },
  "actionConfig": {
    "actionType": "REPLY",
    "content": "亲，这款商品现在活动价xxx元哦~"
  }
}
```

**事件规则**：
```json
{
  "ruleType": "EVENT",
  "triggerConfig": {
    "eventType": "ORDER_PAID"  // ORDER_CREATED/ORDER_PAID/ORDER_SHIPPED
  },
  "actionConfig": {
    "actionType": "NOTIFY",
    "content": "您的订单已支付成功，我们会尽快发货~"
  }
}
```

#### 2.4.3 工作流引擎

**工作流节点类型**：
- 条件节点：条件判断分支
- 动作节点：执行具体操作
- 等待节点：等待外部事件
- 并行节点：并行执行多个分支

### 2.5 CRM客户管理模块

#### 2.5.1 客户等级

| 等级 | 条件 | 权益 |
|------|------|------|
| 普通会员 | 注册用户 | 基础服务 |
| 银卡会员 | 累计消费≥500 | 优先客服 |
| 金卡会员 | 累计消费≥2000 | 专属客服 |
| 白金会员 | 累计消费≥5000 | 优先发货 |
| 钻石会员 | 累计消费≥10000 | 全部权益 |

#### 2.5.2 客户标签

**系统标签**：
- VIP客户
- 新客户
- 复购客户
- 活跃客户
- 沉默客户
- 高价值客户

**自定义标签**：
- 商家可自定义标签
- 支持批量打标签
- 标签支持颜色区分

#### 2.5.3 RFM模型

**RFM评分规则**：
| 维度 | 说明 | 评分方法 |
|------|------|----------|
| R(Recency) | 最近一次购买时间 | 越近分越高 |
| F(Frequency) | 购买频率 | 越高分越高 |
| M(Monetary) | 消费金额 | 越高分越高 |

**客户分类**：
- 重要价值客户：R高F高M高
- 重要发展客户：R高F低M高
- 重要保持客户：R低F高M高
- 重要挽留客户：R低F低M高
- 一般客户：其他
